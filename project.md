# 毕业设计答辩准备文档

## 项目名称：基于 STC8H + MaixCam 的智能视觉测量系统

---

## 一、项目概述（30秒电梯演讲版）

本系统由 **STC8H8K64S4 单片机** 和 **MaixCam AI 视觉模块** 两部分组成，通过串口通信协同工作，实现了以下功能：

1. **INA226 实时功耗监测**：采集被测系统的电流、电压、功率，记录历史最大功率
2. **MaixCam 视觉测量**：利用摄像头+OpenCV 识别目标物体的形状、测量距离和尺寸
3. **LCD4002 数据显示**：将测量结果和功耗数据实时显示在 40x2 字符液晶屏上
4. **4x4 矩阵键盘模式切换**：通过按键选择不同的测量模式

整个系统实现了"按键触发 → 视觉测量 → 结果回传 → LCD 显示"的完整闭环。

## 三、通信协议详解

### 3.1 STC → MaixCam（5字节指令帧）

```
字节0   字节1   字节2   字节3   字节4
0x00    0xFF    Mode    Param   0xFE
帧头1   帧头2   模式号   参数    帧尾
```

- `Mode`：1~4 对应四种测量模式，0 表示空闲
- `Param`：预留参数字节（当前固定为 0x00）

### 3.2 MaixCam → STC（7字节结果帧）

```
字节0   字节1   字节2    字节3    字节4    字节5    字节6
0x00    0xFF    DistH    DistL    LenH     LenL     0xFE
帧头1   帧头2   距离高位  距离低位  长度高位  长度低位  帧尾
```
 
- `Distance`：16位大端序，单位毫米（mm）
- `Length`：16位大端序，单位0.1毫米

### 3.3 STC 端接收状态机（isr.c 中实现）

```
STATE_WAIT_HEADER1 ──0x00──→ STATE_WAIT_HEADER2
STATE_WAIT_HEADER2 ──0xFF──→ STATE_DIST_HIGH
STATE_DIST_HIGH    ──任意──→ STATE_DIST_LOW    (保存 dist_high)
STATE_DIST_LOW     ──任意──→ STATE_LEN_HIGH    (保存 dist_low)
STATE_LEN_HIGH     ──任意──→ STATE_LEN_LOW     (保存 len_high)
STATE_LEN_LOW      ──任意──→ STATE_WAIT_FOOTER (保存 len_low)
STATE_WAIT_FOOTER  ──0xFE──→ 解包成功 → mymode=0（回空闲）
                   ──其他──→ 复位到 STATE_WAIT_HEADER1
```

**关键设计**：收到完整合法帧后，自动将 `mymode=0`，系统回到空闲态。

## 四、四种工作模式详解

### MODE 1 —— 基础图形测量

**功能**：识别参考矩形（A4纸）内的图形（正方形/三角形/圆形），测量距离和图形尺寸

**算法流程**：
1. CLAHE 自适应直方图均衡化 → 双边滤波 → Otsu 二值化 → 形态学闭运算 → Canny 边缘检测
2. 找到参考矩形（A4纸，长宽比 1.3~3.5，面积最小的有效轮廓）
3. 利用针孔相机模型计算距离：`distance = (已知宽度 × 焦距) / 像素宽度`
4. 识别内部图形并计算实际尺寸（通过与参考矩形的像素比例换算）

**图形识别方法**：
- **圆形**：综合5个特征判断 —— 圆形度 > 0.85、圆拟合度 > 0.55、椭圆拟合度 > 0.8、对称性 > 0.9、凸度 > 0.8
- **三角形**：多边形逼近后顶点数为3，判断角度是否接近60°（等边三角形）
- **正方形**：四边形、长宽比 0.9~1.2、角度偏差 < 10°、凸度 > 0.95

**尺寸计算**：`实际尺寸 = (目标像素长度 / 参考矩形像素长度) × 参考矩形实际长度`

### MODE 2 —— 测最小目标物

**功能**：检测重叠矩形中边长最小的正方形

**算法流程**：
1. 高斯模糊 → Canny 边缘检测 → 轮廓检测
2. 找到"内框"（Inner Rect）：面积最小的、长宽比在 1.4~2.5 范围内的长方形
3. **透视变换**：将倾斜的内框校正为标准矩形（993×1500 像素）
4. 在校正后的图像上：Otsu 二值化 → 反色 → 填充孔洞 → 提取最内层轮廓
5. **IOU重叠矩形检测**（`DEBUG_IOUDETECT` 类）：
   - 遍历轮廓上的连续四点组 (p1, p2, p3, p4)
   - 检查两个 90° 角 + 一条已知边 → 几何推算第四个点
   - NMS 去重，保留非重叠的正方形
6. 取所有检测到正方形中**边长最小的**，通过比例换算得到实际尺寸

**关键公式**：`实际长度 = 最小边长像素 / TARGET_HEIGHT × 25.5cm × 10`

### MODE 3 —— 测最大目标物

**功能**：与 MODE 2 使用完全相同的检测流水线，唯一区别是选择策略不同

**区别**：MODE 2 取 `min(边长)`，MODE 3 取 `max(边长)`

**选择逻辑**：
```python
# MODE 2: 取最短边
minAreaLength = detector.DEBUG_GetShortestRectLength()

# MODE 3: 遍历所有矩形取最大平均边长
for quad in rectCoords:
    avg_side = get_rect_avg_side(quad)
    if avg_side > max_side:
        max_side = avg_side
```

### MODE 4 —— 倾斜目标物测量

**功能**：当目标物相对摄像头存在倾斜角度时的测量模式

**核心原理**：利用透视变换将倾斜的目标物"拉正"后再测量，消除透视畸变的影响

**透视变换过程**：
```python
# 1. 检测四边形顶点并排序
src_pts = order_points(inner_pts)  # 左上→右上→右下→左下

# 2. 定义目标矩形（标准视图）
dst_pts = [[0,0], [993,0], [993,1500], [0,1500]]

# 3. 计算变换矩阵并应用
M = cv2.getPerspectiveTransform(src_pts, dst_pts)
warped = cv2.warpPerspective(image, M, (993, 1500))
```

## 五、核心算法原理

### 5.1 距离测量 —— 针孔相机模型

```
distance = (已知物体实际宽度 × 摄像头焦距) / 物体在图像中的像素宽度
```

- 焦距通过校准获得：`focalLength = 1372 像素`（640×480 分辨率下）
- 校准方法：将 A4 纸放在已知距离处，反算焦距
- MODE -1（校准模式）支持触摸屏微调焦距值

### 5.2 透视变换 —— 消除倾斜畸变

**原理**：通过 4 点对应关系，计算 3×3 单应性矩阵 H，将图像从一个平面投影到另一个平面

**参数**：
- 内框实际尺寸：25.7cm × 17cm
- 变换目标尺寸：993 × 1500 像素（竖屏方向）

### 5.3 IOU 重叠矩形检测

**核心思想**：从轮廓的连续角点中，利用几何关系重建被遮挡的正方形

**检测条件**（OneLineTwoAngle 方法）：
1. 连续四点 p1, p2, p3, p4
2. 向量 v1=p2-p1 与 v2=p3-p2 的夹角 ≈ 90°
3. 向量 v2=p3-p2 与 v3=p4-p3 的夹角 ≈ 90°
4. v1 和 v3 方向近似反平行（夹角 ≈ 180°±20°）
5. 中间边 |p2p3| 在合理范围内

**去重**：使用 NMS（非极大值抑制），IoU 阈值 0.5

### 5.4 图形识别 —— 多特征融合

| 特征 | 计算公式 | 用途 |
|------|---------|------|
| 圆形度 | 4π × 面积 / 周长² | 值越接近1越像圆 |
| 圆拟合度 | 轮廓面积 / 外接圆面积 | 与最小外接圆的贴合程度 |
| 凸度 | 轮廓面积 / 凸包面积 | 凹凸程度 |
| 椭圆拟合度 | 轮廓面积 / 拟合椭圆面积 | 圆是特殊椭圆 |
| 对称性 | 1 - 距离方差/均距² | 径向对称程度 |

## 六、硬件设计要点

### 6.1 STC8H8K64S4 主要参数

| 参数 | 值 |
|------|-----|
| 主频 | 44.2368 MHz（内部 IRC） |
| Flash | 64 KB |
| XRAM | 8 KB |
| 封装 | 64 引脚 |

### 6.2 INA226 电流/电压传感器

- **I²C 地址**：0x40（A0/A1 均接 GND）
- **分流电阻**：0.1Ω
- **采样模式**：连续测量，128 次平均，转换时间 1.1ms
- **校准值**：CAL = 56
- **电流精度**：约 1mA/LSB，软件再除以 1.11 做校正
- **功率计算**：`P = I × V`（软件手动计算，未用功率寄存器）

### 6.3 LCD4002 字符屏

- **驱动链路**：STC → 软件I²C → PCF8574（I²C 转并口） → HD44780 控制器 → LCD
- **I²C 地址**：0x27
- **接口模式**：4位数据模式（D4~D7 通过 PCF8574 的 P4~P7 传输）
- **显示布局**：
  - 第0行：模式状态和测量结果（如 "BASIC D:245 L:170"）
  - 第1行：功耗数据（如 "I1.234A V5.012V P6.185W MAX6.185"）
- **注意**：与 INA226 **共用同一条 I²C 总线**（P1.4/P1.5），通过不同设备地址区分

### 6.4 4×4 矩阵键盘

```
[ 1 ][ 2 ][ 3 ][ A ]   ← P2.4 (行0)
[ 4 ][ 5 ][ 6 ][ B ]   ← P2.5 (行1)
[ 7 ][ 8 ][ 9 ][ C ]   ← P2.6 (行2)
[ * ][ 0 ][ # ][ D ]   ← P2.7 (行3)
  ↑    ↑    ↑    ↑
P2.0 P2.1 P2.2 P2.3 (列)
```

- **扫描方式**：逐行拉低，读列检测
- **消抖处理**：5ms 延时消抖 + 等待松开防连发
- **按键映射**：`1`→MODE1, `2`→MODE2, `3`→MODE3, `4`→MODE4

## 七、软件设计要点

### 7.1 STC 端主循环流程

```c
while(1) {
    // 1. 读取 INA226 电流和电压
    current = ina226_get_current() / 1.11;
    vol = ina226_get_bus_voltage();
    power = current * vol;

    // 2. 更新最大功率记录
    if (maxpower <= power) maxpower = power;

    // 3. LCD 定期清屏防残影（每50轮清一次）
    if (lcd_clear_num >= 50) lcd4002_clear();

    // 4. 刷新 LCD 显示（功耗行 + 模式行）

    // 5. 20次键盘快速扫描（每次间隔2ms）
    for (j = 0; j < 20; j++) {
        key = key_scan();
        if (key != KEY_NONE) apply_key(key);
    }

    // 6. 发送当前模式指令到 MaixCam
    send_mode_command();
}
```

### 7.2 MaixCam 端主循环流程

```python
while not app.need_exit():
    img = cam.read()          # 采集 640×480 图像
    img = img.rotate(90)      # 摄像头竖装，旋转90°

    if MODE == -1:   # 校准模式（触摸屏调参）
    elif MODE == 0:  # 休眠
    elif MODE == 1:  # 基础图形测量
    elif MODE == 2 or MODE == 4:  # 透视+重叠检测（最小）
    elif MODE == 3:  # 透视+重叠检测（最大）

    # 每帧都刷新显示
    disp.show(result_img)
```

### 7.3 中断服务程序设计

- **UART1 中断（interrupt 4）**：每收到一个字节，调用 `ParseUART_Protocol()` 状态机逐字节解析
- **状态机优势**：非阻塞式解析，不需要缓冲整帧，节省 RAM
- **接收完成后自动回空闲**：`mymode = 0`，确保 STC 端不会重复发送已完成的指令

## 八、可能被问到的问题及回答

### Q1: 为什么选择 STC8H + MaixCam 的双处理器架构？

**答**：单片机擅长实时控制和外设驱动（键盘扫描、I²C 通信、中断处理），但计算能力不足以运行 OpenCV 图像处理。MaixCam 基于 RISC-V 处理器，运行 Linux 系统，内置 NPU（神经网络加速器），适合图像处理和 AI 推理。两者通过 UART 串口协作，各司其职，发挥各自优势。

### Q2: 串口通信如何保证数据可靠性？

**答**：
1. **固定帧头帧尾**：0x00 0xFF 作帧头，0xFE 作帧尾，状态机严格校验
2. **MaixCam 端重复发送 3 次**：提高传输成功率
3. **状态机自复位**：任何不匹配的字节都会让状态机回到初始状态，不会累积错误
4. **STC 端逐字节中断接收**：不会丢失数据

### Q3: 距离测量的原理是什么？精度如何保证？

**答**：基于**针孔相机模型**，公式为 `D = (W × F) / P`，其中 W 是物体已知实际宽度，F 是焦距（像素），P 是物体在图像中的像素宽度。精度保证措施：
1. 校准模式（MODE -1）支持实时微调焦距值
2. 使用 CLAHE 自适应直方图均衡化增强对比度
3. 双边滤波保边去噪
4. 参考物（A4纸）尺寸精确已知

### Q4: 透视变换解决什么问题？

**答**：当摄像头不是正对目标物时（存在倾斜角度），图像中的矩形会变成梯形，直接测量像素长度会有很大误差。通过透视变换，我们找到图像中四边形的四个顶点，计算出一个 3×3 的变换矩阵，将倾斜的图像"拉正"为标准矩形视图，然后在校正后的图像上进行精确测量。

### Q5: IOU 重叠矩形检测是怎么工作的？

**答**：当多个正方形重叠摆放时，每个正方形的一部分边缘被遮挡。我的算法通过以下步骤检测：
1. 提取图像中所有轮廓的角点
2. 遍历连续的四个角点组合
3. 检查是否满足"两个直角 + 一条已知边 + 反平行方向"条件
4. 如果满足，通过向量几何推算出被遮挡的第四个角点，重建完整正方形
5. 使用 NMS（非极大值抑制）去除重复检测

这种方法的优势是：即使正方形被遮挡 50% 以上，只要暴露出两个相邻角点和连接边，就能重建出完整形状。

### Q6: INA226 的校准值 CAL=56 是怎么确定的？

**答**：INA226 的校准寄存器值通过公式计算：`CAL = trunc(0.00512 / (Current_LSB × R_shunt))`。其中 R_shunt = 0.1Ω 是分流电阻值。理论值约 51，实际使用 56 是根据实测校准后的修正值，同时代码中还额外除以 1.11 作为软件补偿，确保测量精度。

### Q7: 为什么 LCD4002 和 INA226 可以共用一条 I²C 总线？

**答**：I²C 总线是多设备总线协议，每个设备有唯一地址。INA226 的地址是 0x40，LCD4002 通过 PCF8574 扩展芯片连接，地址是 0x27。主机通过地址寻址来区分通信对象。由于主循环中 I²C 操作是顺序执行的（不在中断中调用），不存在总线竞争问题。

### Q8: 圆形和正方形怎么区分？圆形度不是接近的吗？

**答**：单靠圆形度确实容易混淆（正方形的圆形度约 0.785，圆形约 1.0）。所以我用了**5个特征的综合判定**：
- 圆形度 > 0.85（排除正方形）
- 圆拟合度 > 0.55（轮廓贴合外接圆）
- 椭圆拟合度 > 0.8
- 对称性 > 0.9
- 凸度 > 0.8

正方形虽然圆形度达到 0.785，但达不到 0.85 的阈值，且其圆拟合度明显低于圆形。多特征融合大幅提高了区分可靠性。

### Q9: 摄像头为什么要旋转 90 度？

**答**：MaixCam 的摄像头物理安装方向是横向的，但我们的应用场景中目标物（A4纸）是竖向放置的。为了让图像中的目标物呈现正确的竖向方向，需要在代码中对每帧图像旋转 90°。

### Q10: 如果串口数据出错会怎样？

**答**：状态机具有自恢复能力：
- 帧头不匹配 → 立即复位到初始状态
- 帧尾不是 0xFE → 丢弃整帧，复位状态机
- 中间数据字节恰好等于 0x00 → 不会误判，因为只在 STATE_WAIT_HEADER1 状态才检查 0x00
- 最坏情况：丢一帧数据，下一帧正常接收

### Q12: MODE 2 和 MODE 3 的代码有很多重复，为什么不合并？

**答**：这两个模式确实共享了大部分视觉处理流水线（预处理→找内框→透视变换→重叠检测），唯一区别是最后的目标选择策略：MODE 2 取最小、MODE 3 取最大。在项目的后续优化中，可以将公共流水线提取为独立函数，通过策略参数区分。当前保持分开主要是为了比赛时能快速修改某一个模式而不影响另一个。

### Q13: 系统的主要技术难点是什么？

**答**：
1. **重叠矩形的几何重建**：不是简单的轮廓检测，需要从部分可见的角点推算不可见的顶点
2. **透视变换的精度**：四个角点的排序（左上→右上→右下→左下）必须正确，否则变换结果会完全错误
3. **双处理器同步**：STC 和 MaixCam 之间的状态同步，需要确保测量完成后才发送结果，避免数据竞争
4. **多种图形的鲁棒识别**：光照变化、角度倾斜、部分遮挡都会影响形状识别准确率

### Q14: 如果没有检测到目标物会怎样？

**答**：
- MODE 1：如果没找到参考矩形或没识别到内部图形，不会发送串口数据，MaixCam 继续拍摄下一帧
- MODE 2/3：如果没找到 Inner Rect，显示 "No valid Inner Rect"；如果没检测到重叠矩形，`rectCoords` 为空，不发送数据
- STC 端会持续发送模式指令，直到 MaixCam 返回有效结果并将 mymode 置 0

### Q15: 你在项目中遇到的最大挑战是什么？如何解决的？

**建议回答方向**：
- INA226 校准精度问题 → 通过实测对比和软件补偿因子（/1.11）解决
- 重叠矩形检测的误检 → 增加多条件验证（角度 + 边长 + 方向 + 内部区域判断）+ NMS 去重
- 透视变换角点排序错误 → 使用坐标和/差值排序法（sum 最小=左上，diff 最小=右上）
- STC 与 MaixCam 之间的时序同步 → 使用状态机 + 自动回空闲机制

## 十、项目亮点总结

1. **双处理器协同设计**：STC 负责实时控制，MaixCam 负责视觉，各取所长
2. **多模式测量**：支持基础测量、最小/最大目标选择、倾斜补偿四种模式
3. **IOU 重叠检测算法**：创新性地从部分角点重建被遮挡正方形
4. **透视变换校正**：解决倾斜拍摄下的测量畸变问题
5. **完整的标定体系**：MODE -1 校准模式支持触摸屏实时调参
6. **可靠的通信协议**：状态机解析 + 自动回空闲 + 重发机制
7. **实时功耗监测**：INA226 高精度采样 + 最大功率记录
